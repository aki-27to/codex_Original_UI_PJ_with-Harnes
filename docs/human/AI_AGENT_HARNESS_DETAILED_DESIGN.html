AI Agent Harness 詳細設計ガイド

このページは、このリポジトリの AI エージェントが何者かを、 authority docs や machine-readable contracts に入る前に日本語で掴むための導入ガイドです。

まず全体像をここで掴み、そのあとに README.md、 docs/BEGINNER_PATH.md、 docs/DEMO_FLOWS.md、 docs/CAPABILITY_SURFACE.md の順で読むのが最短です。

これは何か

一言でいうと

上位に固定された憲法・権限・停止条件のもとで、 ユーザー依頼を採択可能な成果物へ変換する 憲法拘束型の高自律ワーカーです。

目指していないもの

無制約な自由意思 AI や、何でもできる generic chat shell は目指していません。 主権は外に固定し、その内側だけを強く自律化します。

中心価値は「動けること」単体ではなく、 通してよい成果物かどうかを evidence 付きで返せることです。

最上位目的

最上位目的は、単に会話を返すことではありません。 ユーザー依頼を、最小の HITL で採択可能な成果物へ変換することです。

  • ユーザー依頼の原文に整合している
  • 潜在意図にも整合している
  • authority と permission boundary を破っていない
  • 出荷基準と停止条件を満たしている
  • 改善余地より改善コストが大きい所で止まれる
これは「言われたから何でもやる」ではありません。 「許された範囲で、ユーザーが本当に欲しいものを仕上げる」です。

基本原理

主権は上に置く

憲法、権限境界、停止条件、ship / no-ship gate は人間側に固定します。

境界の内側では強く自律する

分解、再試行、代替案、長時間実行は bounded safety の内側で自律的に進めます。

潜在意図を取りに行く

原文、途中目標、成果物、評価結果を再比較し、字面だけの一致で終わりません。

改善はしてよいが統治は変えない

planner、retrieval、memory handling、skill は改善してよいが、constitution と core gate は固定です。

fail-open ではなく fail-closed

不確実さが危険域に入ったら、前進ではなく stop / hold / escalate を優先します。

成功も失敗も資産化する

失敗は lesson、成功は reusable skill に変換し、次回の worker を強くします。

レイヤー構造

L0-L2 固定層

憲法、目標契約、評価契約。ここは採択される層で、AI が勝手に書き換えてはいけません。

L3-L4 実行層

理解、計画、実行、記憶、runtime、tool、continuity。worker の実働部分です。

L5 改善層

proposal、eval、shadow、gated apply。自己改善は許すが、統治の外へは出しません。

圧縮すると、目的は採択、評価は契約、能力は改善です。

基本ループ

  1. ユーザー依頼を受け取る
  2. 原文と潜在意図を解析する
  3. authority、permission、budget、stop 条件と照合する
  4. bounded な task contract を固定する
  5. subgoal に分解する
  6. tool と memory を使って実行する
  7. 成果物を評価する
  8. 原文・潜在意図・評価結果を再比較する
  9. 必要なら bounded に改善して再実行する
  10. ship するか escalate するかを evidence 付きで決める
  11. lesson と skill を保存する
