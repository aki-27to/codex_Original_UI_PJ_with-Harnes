# web-creative-brief-and-eval

## Purpose

Prevent "looks fine" from replacing an explicit creative brief, reference comparison, and adoption decision.

## Procedure

1. Lock the design brief: target user, page scope, benchmark, anti-benchmark, and success criteria.
2. Evaluate copy quality, information architecture, responsive behavior, accessibility, visual quality, and implementation quality.
3. Capture desktop/mobile/worst-state evidence when browser tooling is available.
4. Connect findings to worker decision and learning surfaces.

## Evidence

- design brief or user taste memory
- reference/benchmark analysis
- screenshots or browser diagnostics
- reviewer/tester verdicts
- implementation verification commands

## Failure Guard

Do not mark design-sensitive work `COMPLETED` without visual evidence and a comparison target.
