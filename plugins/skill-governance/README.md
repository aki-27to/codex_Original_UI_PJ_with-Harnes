# Skill Governance Plugin

Skill Governance packages reusable Codex skills for managing skills themselves.

Use it when a task is about creating, reviewing, promoting, demoting, archiving, or learning from skills. It is not for ordinary code closeout; use Proof / Closeout for that.

## Included Skills

- `skill-creator-master`: create or harden a skill from a repeated task pattern.
- `skill-design-review-codex`: review skill design, trigger quality, output contracts, evidence, and lifecycle boundaries.
- `skill-promotion-governance`: decide keep, promote, demote, rollback, or archive based on evidence.
- `artifact-improvement-learning`: turn verified artifact improvements into reusable learning events.
- `feedback-to-recurrence-patch`: convert user corrections and validation misses into scoped recurrence patches.

## Boundary

The plugin is a distribution package. It does not make generated skills automatically permanent, does not bypass review, and does not replace repository-specific catalogs or policy tests.

Use `references/harnes-adapter-notes.md` when applying this plugin inside the Harnes repo.
