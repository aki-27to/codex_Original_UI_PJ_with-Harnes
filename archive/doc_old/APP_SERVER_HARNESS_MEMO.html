Codex App Server メモ（忘れない用）

あなたが「マジで理解できた」内容を、そのまま固定した版。

0. 30秒で思い出す版

  1. app-server は AI作業マシン（実行本体）。
  2. server.js は Webの受付 + ハーネス。
  3. app_server_cli_harness.js は CLIの受付 + ハーネス。
Web -> server.js -> app-server
CLI -> app_server_cli_harness.js -> app-server

1. 登場人物を3つだけ覚える

本体

app-server

実際にCodexを動かす場所。作業の進捗と完了を返す。

受付

Web/CLIハーネス

入力を受ける。承認・ログ・表示のルールを持つ。

入口

Web画面 / CLI

あなたが操作する場所。中で本体には直接つながない。

2. 1回の依頼で何が起きるか

  1. あなたが依頼を送る。
  2. 受付（ハーネス）が入力を整える。
  3. app-server が作業開始。
  4. 途中経過が返ってくる。
  5. 最後に「完了」または「失敗」が返る。

3. 用語ミニ辞典（ここだけ見ればOK）

用語 意味
thread チャット部屋（会話全体）
turn 1回の依頼
item 途中メッセージ1件（進捗、コマンド結果など）

4. SDKとの使い分け（最小版）

  1. バッチ/CI中心なら SDK
  2. Web/CLIでリアルタイム表示まで作るなら app-server

5. MCPとの違い（1行で）

app-server は実行エンジン、MCP は外部ツール供給元。

似ているのは「呼び出し先になりうる」点だけ。役割は別。

6. 最後に覚える3行

  1. app-server = 実行本体
  2. server.js / CLI harness = 入口 + ルール
  3. 将来の理想 = 共通ハーネス化
